SSD Swappiness Guide

Understanding and optimizing swappiness values for SSD users

Is swappiness=60 OK for SSD?

Short Answer: YES

A swappiness value of 60 is generally acceptable for most SSD users, but lower values are often recommended.

What is swappiness?

Swappiness is a Linux kernel parameter (from 0 to 100) that controls how aggressively the system will swap memory pages out to the swap space.

  • 0: Minimal swapping (avoids swapping as much as possible)
  • 60: Default value on most Linux distributions
  • 100: Aggressive swapping (kernel will swap aggressively)

Check your current swappiness value:

cat /proc/sys/vm/swappiness

SSD Wear Consideration

SSDs have a limited number of write cycles. Excessive swapping can contribute to wear, though modern SSDs are quite durable.

Recommended Swappiness Values

For SSD Users

10-30

Lower values reduce unnecessary write operations, extending SSD lifespan while still providing swap when truly needed.

Default Value

60

Works fine for most users, but may cause more swapping than necessary on systems with adequate RAM.

Not Recommended

80-100

Too aggressive for SSD users. Causes excessive writing that can reduce drive lifespan without performance benefits.

Based on your RAM size:

RAM Size Recommended Swappiness Explanation
8GB or less 30 Balance between RAM preservation and SSD longevity
16GB 15-20 Enough RAM to avoid frequent swapping
32GB or more 10 Plenty of RAM, minimize swapping to protect SSD

Pro Tip

If you have plenty of RAM (16GB+), consider setting swappiness to 10-20. This significantly reduces unnecessary write operations while still providing swap for memory spikes.

How to Change Swappiness

1

Edit sysctl.conf

Open the configuration file with a text editor:

sudo nano /etc/sysctl.conf
2

Add Swappiness Setting

Add this line to the file (example uses value 20):

vm.swappiness=20

Save the file and exit (Ctrl+O, Enter, Ctrl+X).

3

Apply Changes

Reload the configuration without rebooting:

sudo sysctl -p
4

Verify the Change

Confirm the new value is active:

cat /proc/sys/vm/swappiness

No Reboot Needed

The change takes effect immediately. You don't need to reboot your system.

Monitoring Swap Usage

Check current swap usage:

free -h

Look at the "Swap" row to see total, used, and free swap space.

Monitor swap activity in real-time:

watch -n 1 'free -h | grep -i swap'

This will update every second. Press Ctrl+C to stop.

View swap history:

vmstat 5

Shows system statistics every 5 seconds. Look at the "si" (swap in) and "so" (swap out) columns.

Interpreting Swap Usage

Some swap usage is normal. Concern yourself only if the system is constantly swapping even when you have free RAM, which indicates your swappiness might be too high.

Advanced Considerations

SSD Overprovisioning

Modern SSDs have extra space (overprovisioning) that helps distribute writes and extend lifespan. This makes them more resilient to swap-related writes.

SSD Lifespan

Most modern SSDs can withstand hundreds of terabytes written (TBW). Even with swapping, it would take many years to approach these limits.

Performance vs. Longevity

Finding the right balance depends on your usage:

  • For most users: swappiness=20 provides a good balance
  • For memory-intensive tasks: Keep swappiness=30-40 if you frequently max out your RAM
  • For maximum SSD longevity: Set swappiness=10 and ensure you have plenty of RAM

When to Keep Higher Swappiness

If your system regularly uses all available RAM and becomes unresponsive without swapping, you may need a higher value (30-40) even with an SSD.
